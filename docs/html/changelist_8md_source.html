4 ### 1.2.7
5  * Added FEM control signals
6  * Added RAIL_SetPaCTune in order to tune the PA capacitor value for TX and RX.
7  * Fix typo in RAIL_StreamMode_t : PSUEDO_RANDOM_STREAM to PSEUDO_RANDOM_STREAM
8 
9 ### 1.2.6
10  * Added ability to reset demod.
11 
12 ### 1.2.5
13  * Fix register settings for BER testing.
14  * Fix packet buffer for contents larger than 170 bytes.
15 
16 ### 1.2.4
17  * Present worst case RfSense wake period
18 
19 ### 1.2.3
20  * Optimize address filtering code for operation at higher data rates.
21  * Clarified data rate limitations of address filtering.
22 
23 ### 1.2.2
24  * Added RAIL_SetTime() in order to allow the user to change the RAIL
25  timebase with microsecond granularity.
26  * Remove sprurious calls to RAIL_RxRadioStatus with the
27  RAIL_RX_CONFIG_BUFFER_OVERFLOW argument, which were happening when receive
28  was aborted due to events such as address filtering failures.
29  * Fix incorrect PA output power calculations.
30 
31 ### 1.2.1
32  * Allow RAIL_RxStart and RAIL_TxStart to not error if the radio is heading to
33  idle after a call to RAIL_RfIdle(). Application code does not have to wait
34  until the radio is completely idle before calling a receive or transmit
35  operation as long as the radio is heading to idle.
36 
37 ### 1.2.0
38  * Fix IR calibration such that calibration values are more accurate for
39  radio configurations with datarates below 10Kbps.
40  * Fix IR calibration such that the default calibration value is applied
41  when a better calibration value is unable to be calculated.
42  * Added RAIL_CalInit() for calibration initialization.
43  * Added BER test API and structures for diagnostic use -
44  RAIL_BerConfigSet(), RAIL_BerRxStart(), RAIL_BerRxStop(),
45  RAIL_BerStatusGet(), RAIL_BerConfig_t, RAIL_BerStatus_t.
46  * Enable DC Calibration during initialization.
47 
48 ### 1.1.1
49  * Fix CCA timing to prevent invalid CCA failures when starting out of IDLE
50  state.
51  * Fix RX state transitions to TX when ignoring CRC errors. Will properly take
52  the success route when a CRC error occurs while ignoring CRC errors.
53  * Fix TX error state transition after CCA failure.
54 
55 ### 1.1.0
56  * RAIL_RX_CONFIG_INVALID_CRC changed to RAIL_RX_CONFIG_FRAME_ERROR
57  * This now accurately represents the callback flag.
58  * A frame error is either an invalid crc, a frame coding error, or a
59  variable length error. On detection of a frame error, the radio will
60  proceed to the defined error state.
61  * RAIL_SetRxTransitions and RAIL_SetTxTransitions APIs added
62  * Add ability to configure radio to automatically to transition to a
63  given radio state on completion of a receive or transmit event.
64  * RAIL_SetStateTiming API added
65  * State transition times can be configured. The given timestamps will not
66  perfectly match on-air times, due to the RX and TX chain delays, which
67  are dependent on the radio configuration.
68  * RAIL_AddressFilterByFrameType added
69  * If address filtering and frame type length decoding are both enabled,
70  this gives the ability to enable or disable address filtering per frame
71  type
72 
73 ### 1.0.0
74  * Return quarter dBm resolution from RAIL_GetRSSI(). This changes the return
75  value to an int16_t from an int8_t.
76  * Added calibration APIs (\ref Calibration)
77  * The RAIL library will notify the application via callback,
78  RAILCb_CalNeeded(), whenever it detects a calibration is needed.
79  * An API is provided allowing the application to choose when to calibrate
80  the radio, to pass in a known calibration value, or force a calibration
81  to occur.
82  * RAIL_TxPowerSet() returns the power level that can be set which might not
83  match the requested power level.
84 
85 ### Beta 1 - November 16, 2015
86  * Expose EFR32 RF Energy Sensing capability via a RAIL API.
87  * This functionality allows the radio to be woken from sleep by a large
88  amount of energy over the air.
89  * Added new RAIL Timer APIs (\ref System_Timing)
90  * This provides access to a hardware timer running at the RAIL timebase. You
91  can use this to implement simple protocol timing that may be needed by
92  your low level radio code.
93  * This change also adds a new RAILCb_TimerExpired() that must be implemented
94  by your application.
95  * Added APIs to access the bit rate and symbol rate for the current PHY
96  (RAIL_BitRateGet(), RAIL_SymbolRateGet()).
97  * Cleaned up RAIL type names so that they all begin with the RAIL_ prefix.
98  This will require updates to existing application code.
99  * Initial integration with Simplicity Studio's application builder tools
100 
101 ### Alpha 2 - September 14, 2015
102  * Filtering of receive packets based on defined address fields and compare
103  values. See \ref Address_Filtering for more details.
104  * Added different \ref Pre-Transmit options for the Tx function.
105  * Scheduling of packet transmission at an absolute time or after a defined
106  delay.
107  * Packet transmission conditional on Clear Channel Assessment (CSMA, LBT).
108  * The ability to read RSSI manually with RAIL_RxGetRSSI().
109  * Support for building RAIL applications using both GCC and IAR toolchains.
110 
111 ### Alpha 1
112  * Initial release with support for transmit receive and basic radio
113  configuration.
